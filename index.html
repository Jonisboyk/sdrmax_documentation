Onboarding de cliente

processo de ativação · SaaS personalizado · ~5 dias úteis

progresso
0 / 8
01
Contrato fechado manual
Cliente assinou e pagou. Registrar dados iniciais para abertura do processo de ativação.
✓
Receber pagamento / NF emitida
✓
Coletar razão social, CNPJ, contato responsável
✓
Agendar reunião de briefing (48h após pagamento)
02
Briefing SDR manual
Reunião de 1h para entender o negócio, ICP, produto, objeções, tom de voz e objetivo do SDR.
✓
Produto / serviço: o que vende, ticket, diferenciais
✓
ICP: cargo, segmento, dores principais
✓
Objeções mais comuns e como contornar
✓
Tom de voz: formal, casual, técnico?
✓
Objetivo final: agendar reunião? enviar proposta? qualificar?
✓
Horário de funcionamento e fuso
03
Setup WABA + YCloud config
Configurar o número WhatsApp Business do cliente na YCloud e apontar o webhook para a instância do SdrMax.
✓
Criar / verificar conta YCloud do cliente
✓
Registrar número WA Business (WABA ID)
✓
Configurar webhook URL: POST /{client_id}
✓
Testar recebimento de webhook (mensagem de teste)
✓
Configurar chave Gemini (ia_api_key)
04
Criar cliente no banco automático
Registrar o novo client_id no banco, vincular waba_id e ia_api_key. Isso habilita o roteamento correto de todos os webhooks.
✓
INSERT em clients: name, waba_id, ia_api_key, status='setup'
✓
Configurar config JSONB: debounce_ms, max_iter, fuso
✓
Definir blocks JSONB: palavras/frases que encerram IA
✓
Confirmar client_id gerado e documentar internamente
05
Construção do prompt colaborativo
Escrever o system prompt personalizado com base no briefing. Essa é a etapa mais crítica — define 80% da qualidade do SDR.
✓
Redigir system prompt: persona, missão, produto, ICP
✓
Adicionar seção de objeções e respostas
✓
Definir CTA final (ex: "agendar pelo link X")
✓
Escrever 3–5 exemplos de conversa (few-shot)
✓
Salvar prompts JSONB no registro do cliente
06
Teste interno config
Simular conversas reais antes de apresentar ao cliente. Cobrir cenários positivos, objeções e fluxos de erro.
✓
Testar fluxo padrão (lead interessado → CTA)
✓
Testar objeções (preço, concorrente, não tenho tempo)
✓
Testar handoff humano (palavra de bloqueio)
✓
Testar envio de áudio e imagem
✓
Verificar tokens e custo no dashboard
07
Aprovação do cliente colaborativo
Demo ao vivo para o cliente validar o SDR. Coletar feedback e ajustar prompt se necessário.
✓
Agendar call de aprovação (30 min)
✓
Demonstrar 2–3 fluxos ao vivo
✓
Coletar ajustes de tom / resposta
✓
Iterar prompt (máx 2 rodadas)
✓
Obter aprovação formal (mensagem / e-mail)
08
Go live + monitoramento automático
Ativar o cliente em produção. Monitorar os primeiros 7 dias de perto para garantir qualidade e ajustar se necessário.
✓
UPDATE clients SET status = 'active'
✓
Comunicar go live ao cliente
✓
Monitorar dashboard: custo, tokens, conversas (D+1, D+3, D+7)
✓
Revisar qualidade das respostas nos primeiros 50 leads
✓
Agendar check-in semanal com o cliente (30 dias)
